    Quer ver a análise da Assembléia Legislativa de seu estado ou da câmara dos vereadores de seu município no Radar Parlamentar?
    Se você é um desenvolvedor e a casa legislativa de seu interesse oferece dados abertos sobre suas votações, é fácil colocar essa casa no Radar! Basta criar um novo importador!

    A página de análises é capaz de gerar a análise gráfica das casas legislativas em nosso banco de dados. Contudo, os dados de cada casa legislativa possuem origens e formatos diferentes. Por exemplo, para a Câmara dos Deputados, nós acessamos os web services disponíveis, enquanto que para a Câmara Municipal de São Paulo processamos os XMLs que são disponibilizados no site dessa casa. Portanto, para cada casa legislativa pode existir um processo de importação bem diferente dos demais. Outras casas legislativas poderiam disponibilizar dados em arquivos csv, xml, 'bancos de dados' etc.

    No Radar Parlamentar, cada casa legislativa possui um importador associado a ela. Um importador é um módulo na app importdares de nosso sistema Django, que é responsável por importar os dados abertos sobre votações de uma determinada casa legislativa para o nosso banco de dados. Essa estrutura de importador pode ser vista como uma estrutura de "plugin", pois, após um desenvolvedor criar um importador, basta executar esse importador no ambiente de produção do Radar Parlamentar para que a análise da nova casa legislativa seja acessível no site do Radar Parlamentar. Algumas novas linhas também deverão ser acrescentadas aos HTMLs do site apenas para que a análise seja acessível por um menu.

    Para fazer um novo importador, basta criar um módulo em Python que seja capaz de ler os dados abertos da casa legislativa de interesse e escrever os dados em nosso banco de dados (dica: utilizar a API do Django para isso!). A estrutura de nosso banco de dados é definida pelo módulo models, cuja estrutura pode ser também visualizada neste diagrama de classes. Os dados já importados no Radar Parlamentar, separados por casa legislativa, podem ser vistos aqui. Por fim, exemplos de importadores são o importador da Câmara dos Deputados e o importador da Câmara Municipal de São Paulo.

    Quando se for criar um novo importador, é preciso tomar o cuidado para criar um processo de importação incremental. Isso quer dizer que se rodarmos o script com uma fonte de dados na versão v, e depois rodamos o mesmo script de importação com uma fonte de dados atualizada na versão v+1, as seguintes propriedades são desejáveis: 1. os dados importados não devem ser replicados; 2. os novos dados devem ser importados; 3. a importação não deve travar.
